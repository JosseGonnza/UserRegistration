<h1>UserRegistration Kata</h1>

El objetivo de esta kata es implementar un sistema de registro de usuario con varios requisitos de validación, tales como la verificación de contraseñas seguras, la validación de correos electrónicos y la generación de IDs únicos y aleatorios. Cada nueva funcionalidad se debe integrar cuidadosamente en el sistema existente mediante TDD. Esto implica escribir primero una prueba que falle, luego el código mínimo necesario para pasar la prueba, y finalmente refactorizar el código para mejorar su estructura sin cambiar su comportamiento. Este método garantiza que el sistema crezca de manera controlada y sólida, mejorando con cada iteración.
___
### Pasos a seguir:
-  Devuelve el usuario recién creado.
- La contraseña debe cumplir los requisitos de seguridad.
-  Tener más de 8 caracteres.
-  Contener un guión bajo.
-  El Id de usuario debe ser aleatorio y único.
-  El email debe ser un email válido.
-  El correo electrónico no puede ser utilizado por otro usuario.
-  Se envía un correo electrónico de bienvenida al usuario recién registrado.
